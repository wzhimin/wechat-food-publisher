# 微信小程序后台管理系统 — 实现文档

> 最后更新：2026-04-20

## 一、系统架构

### 技术栈
- **后端**：Express.js + Sequelize ORM + MySQL
- **前端**：纯 HTML + 原生 JS + Bootstrap 5 CDN（无构建工具）
- **认证**：账号密码登录（非微信 OpenID），Session 存储

### 登录信息
- 默认管理员：`admin` / `wang123456`
- 登录后通过 `/api/admin/login` 获取 token，后续请求 `Authorization: Bearer <token>`

## 二、数据库扩展

### 已执行的迁移（admin-migration.sql）

| 表 | 新增字段 | 说明 |
|---|---------|------|
| `recipe_comments` | `status` ENUM('pending','approved','rejected') DEFAULT 'pending' | 评论审核状态 |
| `recipes` | `status` ENUM('pending','approved','rejected') DEFAULT 'approved' | 菜谱审核状态 |
| `recipes` | `is_featured` TINYINT(1) DEFAULT 0 | 精选标志 |
| `feedbacks` | `admin_reply` TEXT | 管理员回复 |
| `feedbacks` | `handled_by` VARCHAR(64) | 处理人 |
| `feedbacks` | `handled_at` DATETIME | 处理时间 |

### 新增表

**reports（举报表）**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| type | ENUM('recipe','comment') | 举报类型 |
| recipeId | INT | 关联菜谱 |
| commentId | INT | 关联评论 |
| reason | VARCHAR(100) | 举报原因 |
| description | TEXT | 详细描述 |
| reporterOpenid | VARCHAR(128) | 举报人 |
| status | ENUM('pending','handled','dismissed') | 处理状态 |
| handled_by | VARCHAR(64) | 处理人 |
| handled_at | DATETIME | 处理时间 |

### Sequelize 同步模式
模型使用 `alter: 'safe'` 模式同步。**注意**：此模式下添加新字段时不能同时创建 indexes，否则部署失败。

## 三、API 端点一览

所有管理 API 以 `/api/admin` 开头，需在 Header 携带 `Authorization: Bearer <token>`。

### 认证
```
POST /api/admin/login          # 登录 → { token, admin: { username } }
```

### 数据统计
```
GET  /api/admin/stats          # 仪表盘数据（总菜谱/用户/评论/反馈）
GET  /api/admin/users/stats    # 用户统计（总数/今日/活跃7天/贡献者）
```

### 评论管理
```
GET    /api/admin/comments             # 评论列表 (?status=&page=&pageSize=)
DELETE /api/admin/comments/:id         # 删除评论
```

### 菜谱管理
```
GET    /api/admin/recipes              # 菜谱列表 (?status=&page=&pageSize=&search=)
PUT    /api/admin/recipes/:id          # 编辑菜谱
DELETE /api/admin/recipes/:id          # 删除菜谱
POST   /api/admin/recipes/feature/:id  # 切换精选状态
DELETE /api/admin/recipes/batch        # 批量删除 (body: { ids: [] })
```

### 用户管理
```
GET  /api/admin/users/list     # 用户列表 (?page=&pageSize=)
```
> **注意**：User 模型无 `last_login` 字段，统计活跃用户用 `updated_at` 替代。

### 反馈管理
```
GET    /api/admin/feedbacks           # 反馈列表 (?status=&page=&pageSize=)
POST   /api/admin/feedbacks/:id/reply # 回复反馈 (body: { reply: '' })
DELETE /api/admin/feedbacks/:id       # 删除反馈
```

### 举报管理
```
GET   /api/admin/reports              # 举报列表 (?status=&page=&pageSize=)
POST  /api/admin/reports/:id/handle   # 处理举报 (body: { status, note })
```

### 非菜谱清理
```
GET    /api/admin/non-dishes          # 疑似非菜谱列表
DELETE /api/admin/non-dishes          # 批量删除 (body: { ids: [] })
```
> **注意**：RecipeNote 无 `recipeId` 字段，批量删除时需排除该模型。

### 图片上传
```
POST /api/upload-image          # 上传图片 (body: { imageBase64: '' }) → { success, url }
```

## 四、前端页面

路径：`public/admin/`

| 页面 | 文件 | 功能 |
|------|------|------|
| 仪表盘 | index.html | 数据概览（菜谱/用户/评论/反馈统计） |
| 评论管理 | comments.html | 查看/删除评论，按状态筛选 |
| 菜谱管理 | recipes.html | 编辑/删除/设精选/批量删除，支持图片上传封面 |
| 反馈管理 | feedbacks.html | 查看/回复/删除反馈 |
| 用户管理 | users.html | 用户列表（OpenID/昵称/头像/注册时间） |
| 非菜谱清理 | non-dishes.html | 批量检测和删除非菜谱条目 |
| 举报管理 | reports.html | 查看/处理举报（设为已处理/驳回） |

### 通用设计
- 所有页面共享 `admin.css` 和 `admin-auth.js`
- 侧边栏导航，统一登录态校验
- 编辑弹窗：菜谱编辑支持封面 URL 输入 + 本地图片上传（→ `/api/upload-image`）

## 六、内容审核策略

| 内容类型 | 策略 | 说明 |
|---------|------|------|
| 用户发布菜谱 | 自动通过 | `status='approved'`，依赖举报机制兜底 |
| 用户评论 | 自动通过 | 同上 |
| 图片 | img_sec_check | 发布时调用微信图片安全检测 |

### 图片安全检测
后端在菜谱发布/更新时，如有新上传图片，调用微信 `img_sec_check` 接口检测。违规图片拒绝发布。
