# 微信小程序后台管理系统设计方案

## 一、当前系统现状

### 已有的表结构
| 模型 | 字段 | 审核状态 |
|------|------|---------|
| Recipe | id, title, cover, ingredients, steps, author, etc. | ❌ 无审核字段 |
| RecipeComment | id, recipe_id, content, openid, created_at | ❌ 无审核字段 |
| RecipeLike | id, recipe_id, openid, created_at | - |
| RecipeNote | id, recipe_id, openid, content, created_at | ❌ 无审核字段 |
| Feedback | id, type, content, contact, status, created_at | ⚠️ 有 status 但无管理 API |
| User | id, openid, nickname, avatar, role | ⚠️ 有 role 字段 |

### 当前 API 问题
- ❌ 无管理后台 API（评论审核、菜谱审核、反馈管理）
- ❌ 反馈只有用户自己查看的 API，没有管理员查看
- ❌ 评论无审核，发布即显示

## 二、需要扩展的功能

### 1. 数据库字段扩展

```sql
-- 评论表增加审核字段
ALTER TABLE recipe_comments ADD COLUMN status ENUM('pending', 'approved', 'rejected') DEFAULT 'pending';

-- 菜谱表增加审核字段
ALTER TABLE recipes ADD COLUMN status ENUM('pending', 'approved', 'rejected') DEFAULT 'pending';
ALTER TABLE recipes ADD COLUMN is_featured TINYINT(0) DEFAULT 0;

-- 反馈表增加管理员字段
ALTER TABLE feedbacks ADD COLUMN admin_reply TEXT;
ALTER TABLE feedbacks ADD COLUMN handled_by VARCHAR(50);
ALTER TABLE feedbacks ADD COLUMN handled_at DATETIME;
```

### 2. 新增管理 API

```
GET    /api/admin/comments?status=pending    # 评论列表（可筛选）
POST   /api/admin/comments/:id/approve       # 通过评论
POST   /api/admin/comments/:id/reject         # 拒绝评论

GET    /api/admin/recipes?status=pending     # 菜谱列表（待审核）
POST   /api/admin/recipes/:id/approve         # 通过菜谱
POST   /api/admin/recipes/:id/reject         # 拒绝菜谱
POST   /api/admin/recipes/:id/feature        # 设为精选

GET    /api/admin/feedbacks?status=pending  # 反馈列表
POST   /api/admin/feedbacks/:id/reply        # 回复反馈
POST   /api/admin/feedbacks/:id/resolve      # 标记已处理

GET    /api/admin/stats                      # 数据统计
```

## 三、后台页面设计

### 页面结构

```
/admin/
├── index.html          # 仪表盘（数据概览）
├── comments.html       # 评论审核
├── recipes.html        # 菜谱管理（含用户发布）
├── feedbacks.html      # 反馈管理
├── users.html          # 用户管理（可选）
└── settings.html      # 系统设置
```

### 功能模块

#### 1. 仪表盘 (index.html)
- 今日数据：新增菜谱、新增评论、新增反馈
- 待审核：待审核评论数、待审核菜谱数
- 数据趋势：近7天数据图表

#### 2. 评论审核 (comments.html)
- 列表：显示评论内容、所属菜谱、用户、时间
- 操作：批准、拒绝、删除
- 筛选：全部、待审核、已通过、已拒绝

#### 3. 菜谱管理 (recipes.html)
- 列表：显示标题、封面、作者、发布时间、状态
- 操作：预览、通过、拒绝、设为精选、删除
- 筛选：全部、待审核、已通过、已拒绝、精选

#### 4. 反馈管理 (feedbacks.html)
- 列表：显示类型、内容、联系方式、状态、时间
- 操作：查看详情、回复、标记已处理
- 筛选：全部、待处理、已处理

## 四、技术实现

### 1. 管理员权限控制

方案A：简单实现（推荐）
- 在 User 表增加 `role` 字段：`admin` / `user`
- 登录时通过微信 openid 识别管理员
- 管理 API 检查 role=admin

方案B：JWT 方案
- 增加登录 API，返回 token
- 管理 API 检查 token 中的 role

### 2. 前端页面

纯 HTML + Bootstrap + Vue CDN，无需构建：
- Bootstrap 5（CDN）：UI 框架
- Vue 3（CDN）：数据绑定
- Axios（CDN）：API 调用
