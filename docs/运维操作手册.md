## 二、数据库操作

### 连接数据库
```bash
mysql -u root -p nodejs_demo
```

### 常用查询
```sql
-- 查看菜谱总数和状态分布
SELECT status, COUNT(*) FROM recipes GROUP BY status;

-- 查看精选菜谱
SELECT id, title FROM recipes WHERE is_featured = 1;

-- 查看近期举报
SELECT * FROM reports WHERE status = 'pending' ORDER BY created_at DESC LIMIT 20;

-- 查看用户统计
SELECT COUNT(*) as total_users FROM users;
SELECT COUNT(*) as today_users FROM users WHERE created_at >= CURDATE();
SELECT COUNT(*) as active_users FROM users WHERE updated_at >= DATE_SUB(NOW(), INTERVAL 7 DAY);

-- 查看系统导入菜谱
SELECT COUNT(*) as system_recipes FROM recipes WHERE authorOpenid = 'system';
SELECT COUNT(*) as user_recipes FROM recipes WHERE authorOpenid != 'system';
```

### 数据库迁移
新增字段通过 Sequelize `alter: 'safe'` 模式自动同步，无需手动执行 SQL。
如需手动迁移，SQL 文件在 `docs/admin-migration.sql`。

## 四、运维脚本

### 1. 封面补全脚本
为无封面菜谱自动搜索 Pixabay 图片并补全。

```bash
node scripts/fill-recipe-covers.js
```

### 2. 封面失效检测脚本
检测第三方封面（Pixabay/Pexels CDN 链接）是否失效。

```bash
# 仅检测
node scripts/check-pixabay-covers.js

# 检测并自动修复失效封面
node scripts/check-pixabay-covers.js --fix
```

建议：每周运行一次检测，或设置 cron 定时任务。

### 3. 非菜谱清理脚本
扫描数据库中疑似非菜谱的条目（纯食材名、过于简短的标题等）。

```bash
node scripts/clean_non_dish.js
```

## 六、常见问题

### Q: 后台登录提示"用户名或密码错误"
检查环境变量 `ADMIN_USERNAME` 和 `ADMIN_PASSWORD` 是否正确设置，重启服务后生效。

### Q: 图片上传失败
检查 `/api/upload-image` 接口是否正常，确认服务器磁盘空间充足，上传目录有写权限。

### Q: Pixabay 封面显示 403/400
Pixabay CDN 链接有时效性。运行 `check-pixabay-covers.js --fix` 批量替换失效封面。

### Q: 小程序用户反馈收不到
检查云函数配置和反馈表。反馈管理入口在后台 `/admin/feedbacks.html`。

### Q: Sequelize 同步失败 "Unknown column" 或索引冲突
模型定义新增字段时，如有 indexes 定义，在 `alter: 'safe'` 模式下可能失败。临时移除 indexes 再部署。
